# main-portal-journal

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存储，不依赖特定后端服务或数据库，可直接通过 Web 服务器或 GitHub Pages 访问。

## 项目简介

`main-portal-journal` 是一个静态页面集合仓库，主要用途包括：

- 存档各类独立制作的 HTML 页面
- 作为页面内容的集中发布源
- 提供干净、可维护的目录结构以便管理多个页面

仓库本身不绑定任何具体域名，也不针对单一网站。所有页面均为独立实体，彼此之间无强制关联。

## 目录说明

```
.
├── index.html        # 仓库首页（可选，可自行修改或删除）
├── pages/            # 存放所有独立 HTML 页面
│   ├── example1/
│   ├── example2/
│   └── ...
├── assets/           # 公共资源（CSS、JS、图片等）
├── README.md         # 本文件
└── .gitignore        # Git 忽略规则
```

- `pages/`：每个子目录对应一个独立页面，内部包含该页面所需的 HTML 及专属资源。
- `assets/`：存放多个页面可能共用的样式、脚本或图像文件，避免重复。

## 页面归档说明

- 每个页面应保持独立，不依赖其他页面内容。
- 页面内部可引用 `assets/` 下的公共资源，也可自带私有资源。
- 建议为每个页面编写简短的内部说明（如注释或独立的 README），方便后续维护。
- 归档时请确保页面结构完整，无断链或缺失文件。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录，放入 HTML 及相关文件。
- 更新页面：直接修改对应目录下的文件，保留历史版本（通过 Git 管理）。
- 删除页面：移除对应目录，并检查是否有公共资源引用需要清理。
- 请勿在仓库中存放与页面内容无关的大文件（如视频、压缩包等），以免影响克隆与部署效率。

## 部署

本仓库可直接启用 GitHub Pages 进行静态站点部署。部署后，每个页面可通过 `https://<你的用户名>.github.io/main-portal-journal/pages/<页面目录>/` 访问。

如需自定义首页，可修改根目录下的 `index.html`。

## 许可

本仓库内容默认不附带特定许可证。如页面中包含第三方资源，请遵守其原始许可条款。
